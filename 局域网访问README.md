# 🌐 智能门店看板 - 局域网访问快速指南

## 📋 问题现象
同局域网下的其他设备无法访问智能门店看板系统

---

## ⚡ 快速解决（3步搞定）

### 第1步：配置防火墙
**右键点击** → **以管理员身份运行**：
```
一键配置局域网访问.bat
```

### 第2步：启动看板
双击运行：
```
启动看板.bat
```

### 第3步：获取访问地址
启动后在控制台查看显示的局域网地址，例如：
```
🌐 局域网访问: http://192.168.1.100:8050
```

---

## 🔍 诊断工具

如果上述步骤完成后仍然无法访问，运行诊断工具：
```
诊断局域网访问.bat
```

诊断工具会自动检查：
- ✅ 管理员权限
- ✅ 本机IP地址
- ✅ 防火墙规则
- ✅ 端口状态
- ✅ 网络类型

## ❓ 常见问题

### Q1: 为什么需要管理员权限？
A: 配置Windows防火墙需要管理员权限。只需在配置时运行一次，之后正常启动看板不需要管理员权限。

### Q2: 配置后还是无法访问怎么办？
A: 按顺序检查：
1. 运行 `诊断局域网访问.bat`
2. 检查其他设备是否连接到同一WiFi
3. 尝试Ping本机IP测试网络连通性
4. 临时关闭杀毒软件测试
5. 查看详细故障排查指南

### Q3: 显示的IP地址是127.0.0.1怎么办？
A: 说明未检测到局域网IP，请检查：
- 是否已连接WiFi或有线网络
- 网络适配器是否正常工作
- 运行 `ipconfig` 查看网络配置

### Q4: 可以用手机访问吗？
A: 可以！确保手机连接到同一WiFi，然后在手机浏览器输入局域网地址即可。

## 🛠️ 技术细节

### 监听配置
```python
# 智能门店看板_Dash版.py
app.run(
    host='0.0.0.0',  # 监听所有网络接口
    port=8050        # 端口号
)
```

### 防火墙规则
```powershell
# 允许入站连接到8050端口
New-NetFirewallRule `
    -DisplayName "智能门店看板" `
    -Direction Inbound `
    -Protocol TCP `
    -LocalPort 8050 `
    -Action Allow
```
