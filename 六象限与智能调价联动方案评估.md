## 📋 Part 1: 业务运营视角评估

### 1.1 当前业务流程分析

#### 现状流程（断裂的）
```
步骤1：商品健康分析
  ↓ 发现问题
  用户看到：💎 潜力商品 180个
  
步骤2：人工判断
  ↓ 用户思考
  "这些商品该怎么办？"
  
步骤3：切换到调价计算器
  ↓ 手动操作
  重新筛选、重新分析
  
步骤4：制定调价方案
  ↓ 执行
  导出、上传平台
```

**问题**：
- ❌ 流程断裂，需要手动切换
- ❌ 重复筛选，效率低下
- ❌ 缺少上下文，决策困难

#### 理想流程（联动的）
```
步骤1：商品健康分析
  ↓ 发现问题
  用户看到：💎 潜力商品 180个
  
步骤2：一键联动
  ↓ 自动跳转
  点击"调价优化" → 自动带入潜力商品数据
  
步骤3：智能调价建议
  ↓ 系统分析
  基于象限特征，给出针对性建议
  
步骤4：执行调价
  ↓ 导出执行
  一键导出调价方案
```

**优势**：
- ✅ 流程连贯，一气呵成
- ✅ 自动筛选，节省时间
- ✅ 上下文保留，决策精准

### 1.2 六象限与调价策略的匹配度

#### 🌟 明星商品（高利润+高动销+高价值）

**业务特征**：
- 又赚钱又好卖
- 门店核心盈利品
- 客户认可度高

**调价策略**：
- ✅ **测试性提价**：评估价格弹性上限
- ✅ **保持现价**：如果已经是最优价格
- ❌ **不建议降价**：会损失利润

**联动价值**：⭐⭐⭐⭐⭐（非常高）
- 明星商品是最适合提价的
- 需要精确计算价格弹性
- 避免提价过度导致销量下滑

**调价计算器支持**：
```python
# 当前调价计算器的"明星款"定义
if high_profit and high_sales and not high_marketing:
    return ('明星款', '🌟', '核心商品，可小幅提价', True, 'optional')
```
✅ **匹配度高**：调价计算器已有"明星款"角色

---

#### 🔥 畅销商品（低价+高销+正利润）

**业务特征**：
- 刚需基础品（包子、矿泉水）
- 价格敏感度高
- 带动客流

**调价策略**：
- ⚠️ **谨慎提价**：可能影响客流
- ✅ **维持现价**：保持竞争力
- ❌ **不建议降价**：已经是低价

**联动价值**：⭐⭐⭐（中等）
- 畅销商品调价风险大
- 需要对标竞品价格
- 更适合"价格监控"而非"主动调价"

**调价计算器支持**：
```python
# 当前调价计算器的"走量款"定义
if not high_profit and profit_rate >= 5 and high_sales:
    return ('走量款', '⚡', '薄利多销，建议提价', True, 'up')
```
⚠️ **匹配度中等**：走量款建议提价，但畅销商品应谨慎

---

#### 💎 潜力商品（高利润+低动销+有价值）

**业务特征**：
- 利润好但销量低
- 需要增加曝光
- 可能是定价过高

**调价策略**：
- ✅ **测试性降价**：评估价格敏感度
- ✅ **促销活动**：配合营销推广
- ⚠️ **小幅提价**：如果是高端定位

**联动价值**：⭐⭐⭐⭐⭐（非常高）
- 潜力商品最需要调价优化
- 降价可能带来销量突破
- 需要精确计算ROI

**调价计算器支持**：
```python
# 当前调价计算器的"潜力款"定义
if high_profit and not high_sales:
    return ('潜力款', '💎', '利润好销量低，可提价', True, 'optional')
```
❌ **匹配度低**：调价计算器建议提价，但潜力商品更应该降价促销！

---

#### 🎯 策略引流（极端价格引流品）

**业务特征**：
- 主动亏损引流
- 秒杀/满赠活动
- 监控ROI

**调价策略**：
- ❌ **不建议调价**：这是策略性定价
- ✅ **监控效果**：评估引流ROI
- ✅ **控制频率**：避免常态化

**联动价值**：⭐（很低）

#### 🐌 低效商品（低利润+低动销）

## 📊 Part 2: 数据分析视角评估

### 2.1 数据逻辑一致性分析

#### 问题1：六象限 vs 调价角色的定义差异

**六象限定义**（基于商品健康分析）：
```python
# 明星商品
high_profit = profit_rate > profit_threshold  # 品类中位数
high_sales = is_high_sales(sales_index, sales_qty, order_count)  # 70分位数
high_value = unit_profit >= 0.5 or total_profit >= 50
```

**调价角色定义**（基于销售数据）：
```python
# 明星款
high_profit = profit_rate > 20  # 固定20%
high_sales = sales_percentile >= 0.7  # 前30%
high_marketing = marketing_ratio > 10  # 营销占比
```

**差异分析**：
| 维度 | 六象限 | 调价角色 | 一致性 |
|------|--------|---------|--------|
| 利润率阈值 | 品类中位数（动态） | 固定20% | ❌ 不一致 |
| 销量阈值 | 70分位数+订单数 | 前30%（70分位数） | ✅ 基本一致 |
| 价值判断 | 单品利润≥0.5元 | 无 | ⚠️ 部分一致 |
| 营销维度 | 无 | 营销占比>10% | ⚠️ 维度不同 |

**结论**：
- ⚠️ **定义不完全一致**，需要统一或映射
- ✅ **销量维度基本一致**
- ❌ **利润率阈值差异较大**

---

#### 问题2：潜力商品的调价方向矛盾

**六象限判断**：
```
💎 潜力商品 = 高利润 + 低动销 + 有价值
→ 业务含义：利润好但销量低，需要增加曝光
→ 运营策略：上活动、降价促销、关联推荐
```

**调价计算器判断**：
```
💎 潜力款 = 高利润 + 低动销
→ 调价建议："利润好销量低，可提价"
→ 调价方向：optional（可选）
```

**矛盾分析**：
- ❌ **方向相反**：六象限建议降价促销，调价计算器建议提价
- ⚠️ **逻辑冲突**：销量已经低了，提价会更低
- ✅ **可能场景**：高端定位商品，提价提升品牌形象

**解决方案**：
1. **区分场景**：高端商品可提价，普通商品应降价
2. **增加判断**：基于价格区间（高价/中价/低价）
3. **用户选择**：提供两种策略，让用户选择
