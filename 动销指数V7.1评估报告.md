# 动销指数V7.1评估报告

## 变更时间
2024年

## 变更内容

### 旧版本（V5.0）
```
动销指数 = 标准化销量×50% + 标准化周转率×30% + 标准化订单数×20%
```

### 新版本（V7.1）
```
动销指数 = 标准化销量×60% + 标准化订单数×40%
```

### 变更原因
1. **周转率计算不准确**：订单数据无法准确计算库存周转率（库存快照不连续）
2. **简化逻辑**：销量+订单数已经足够评估动销情况
3. **提高可靠性**：移除不准确的指标，避免误导决策

---

## 新版动销指数评估

### 1. 逻辑合理性分析

#### 1.1 销量（60%权重）

**含义**：30天内商品的总销售件数

**优点**：
- ✅ 直观反映商品的销售规模
- ✅ 数据准确，来自订单累计
- ✅ 是最核心的动销指标

**局限**：
- ⚠️ 无法区分"单笔大单"和"多次小单"
- ⚠️ 无法反映"相对于库存，卖得快不快"

**案例**：
```
商品A：销量100件
商品B：销量50件
→ 从销量看，A比B好

但如果：
商品A：100件来自1个订单（团购）
商品B：50件来自20个订单（日常复购）
→ 实际B可能更受欢迎
```

#### 1.2 订单数（40%权重）

**含义**：30天内购买该商品的订单数量（去重）

**优点**：
- ✅ 反映购买频次和受欢迎程度
- ✅ 防止"单笔大单"误判
- ✅ 数据准确，来自订单去重计数

**局限**：
- ⚠️ 无法区分订单大小（1件的订单和100件的订单权重相同）

**案例**：
```
商品A：销量100件，订单数5单 → 平均每单20件（可能是团购）
商品B：销量50件，订单数20单 → 平均每单2.5件（日常复购）

动销指数：
商品A：标准化销量高，但标准化订单数低 → 综合得分中等
商品B：标准化销量中等，但标准化订单数高 → 综合得分也中等

→ 两者动销指数接近，但B更受日常欢迎
```

#### 1.3 权重分配（60% vs 40%）

**为什么是60% vs 40%？**

**方案对比**：
```
方案1：50% vs 50%（平均）
- 优点：简单，两者同等重要
- 缺点：销量是最核心指标，应该更重要

方案2：70% vs 30%
- 优点：销量权重更高
- 缺点：订单数权重太低，防团购效果减弱

方案3：60% vs 40%（当前）✅
- 优点：销量为主，订单数为辅，平衡合理
- 缺点：无明显缺点
```

**结论**：60% vs 40%是合理的权重分配

### 2. 与旧版本对比

#### 2.1 相关性分析

**假设场景**：100个商品的动销数据

| 商品类型 | 销量 | 订单数 | 周转率 | V5.0动销指数 | V7.1动销指数 | 差异 |
|---------|------|--------|--------|-------------|-------------|------|
| 爆款（高销高频） | 200件 | 50单 | 3次 | 0.85 | 0.88 | +0.03 |
| 团购（高销低频） | 150件 | 5单 | 2次 | 0.65 | 0.58 | -0.07 |
| 日常（中销中频） | 80件 | 25单 | 1.5次 | 0.55 | 0.60 | +0.05 |
| 低频（低销低频） | 20件 | 3单 | 0.5次 | 0.15 | 0.12 | -0.03 |

**观察**：
1. ✅ 爆款商品：V7.1得分略高（更准确）
2. ✅ 团购商品：V7.1得分降低（更合理，因为订单数少）
3. ✅ 日常商品：V7.1得分略高（更合理，因为订单数多）
4. ✅ 低频商品：V7.1得分略低（差异不大）

**结论**：V7.1更能区分"真正的高动销"和"团购虚高"

#### 2.2 六象限分类影响

**预估影响**：
- 5-10%的商品可能改变象限分类
- 主要影响：团购类商品（高销低频）可能从"高动销"降为"低动销"

**具体影响**：
```
场景1：团购商品
- 旧版：销量150件，订单5单，周转率2次 → 动销指数0.65（高动销）
- 新版：销量150件，订单5单 → 动销指数0.58（可能变为低动销）
- 影响：从"自然引流"变为"潜力商品"或"低效商品"
- 评价：✅ 更合理，因为团购不代表真正的高动销

场景2：日常复购商品
- 旧版：销量80件，订单25单，周转率1.5次 → 动销指数0.55（中等）
- 新版：销量80件，订单25单 → 动销指数0.60（中等偏高）
- 影响：可能从"低动销"变为"高动销"
- 评价：✅ 更合理，因为高频复购说明受欢迎
```

### 3. 潜在问题和建议

#### 3.1 潜在问题

**问题1：无法反映"相对于库存，卖得快不快"**

**影响**：
```
商品A：销量100件，库存10件 → 周转率10次（超快）
商品B：销量100件，库存500件 → 周转率0.2次（超慢）

V7.1动销指数：
- 如果订单数相同，两者动销指数相同
- 但实际A明显更畅销（库存快卖完了）
```

**建议**：
- 可以考虑增加"库存预警"功能，单独提示库存不足的商品
- 或者在商品详情中显示"库存周转天数"作为参考

**问题2：订单数可能被刷单影响**

**影响**：
```
如果有刷单行为：
- 销量可能虚高
- 订单数也可能虚高
- 动销指数会被拉高
```

**建议**：
- 增加异常订单检测（如：同一用户多次下单、订单金额异常等）
- 在数据清洗阶段剔除异常订单

#### 3.2 优化建议

**建议1：增加"平均每单件数"作为辅助指标**

```python
product_data['平均每单件数'] = product_data['销量'] / product_data['订单数']

# 用于识别团购商品
product_data['是否团购'] = product_data['平均每单件数'] > 10
```

**用途**：
- 在商品详情中显示，帮助运营判断
- 可以作为"策略引流"的辅助判定条件

**建议2：增加"动销稳定性"指标**

```python
# 如果有每日数据，可以计算销量的标准差
# 标准差越小，说明销量越稳定
```

**用途**：
- 区分"稳定畅销"和"偶尔爆发"
- 帮助运营做库存规划

### 4. 总体评估

#### 4.1 优点
1. ✅ **逻辑清晰**：只用销量和订单数，简单易懂
2. ✅ **数据准确**：不依赖不准确的库存数据
3. ✅ **防止误判**：订单数能有效防止团购虚高
4. ✅ **权重合理**：60% vs 40%平衡了规模和频次

#### 4.2 缺点
1. ⚠️ **失去库存维度**：无法反映"相对于库存，卖得快不快"
2. ⚠️ **可能被刷单影响**：需要增加异常订单检测

#### 4.3 总体评分
- **准确性**：★★★★☆（4/5）- 比旧版更准确
- **可靠性**：★★★★★（5/5）- 不依赖不准确的数据
- **实用性**：★★★★☆（4/5）- 能满足大部分需求
- **可解释性**：★★★★★（5/5）- 逻辑简单清晰

**综合评分**：★★★★☆（4.5/5）

## 结论

### V7.1动销指数是一个合理的优化方案

**核心优势**：
1. 移除了不准确的周转率，提高了可靠性
2. 销量+订单数的组合能有效评估动销情况
3. 权重分配合理，平衡了规模和频次

**建议**：
1. ✅ **立即采用**：V7.1是比V5.0更好的方案
2. 💡 **后续优化**：考虑增加"平均每单件数"作为辅助指标
3. 💡 **数据清洗**：增加异常订单检测，防止刷单影响

**预期影响**：
- 5-10%的商品可能改变象限分类
- 主要影响团购类商品（更合理）
- 整体分类准确性提升
