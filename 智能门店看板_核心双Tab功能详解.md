## 看板整体架构

### 系统定位

**智能门店看板是一个数据驱动的门店经营诊断系统**，通过多维度数据分析，帮助运营者：
- 🔍 快速发现经营问题
- 📊 深入分析问题根因
- ✅ 获得可执行的优化建议
- 📈 持续跟踪改进效果

### 核心双Tab架构

```
┌─────────────────────────────────────────────────────────────┐
│  🏪 智能门店看板                                             │
├─────────────────────────────────────────────────────────────┤
│  [数据源选择] 数据库/Excel上传                               │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────┬──────────┐                                    │
│  │✅今日必做│📊订单概览│                                    │

**设计理念**：
- **✅ 今日必做**：问题优先级管理，快速定位紧急问题
- **📊 订单概览**：经营健康度仪表盘，全面掌控业绩指标

## Tab 0: 今日必做

### 功能定位
**问题优先级管理中心** - 自动识别最紧急的经营问题，按优先级分层展示

### 核心功能架构

#### 🔴 紧急处理层（今日必须完成）

**1. 满减穿底订单分析**
- **识别逻辑**：满减优惠 > 订单毛利
- **数据来源**：订单聚合数据（order_agg）
- **分析维度**：
  - 穿底订单数量及占比
  - 穿底订单总亏损金额
  - 单均亏损金额
  - 渠道分布
- **业务影响**：直接亏损，每单都在赔钱
- **行动建议**：立即调整满减规则或商品定价
- **详情功能**：
  - 订单级穿底明细（订单ID、金额、亏损额）
  - 商品级穿底分析（哪些商品导致穿底）

**2. 高配送费订单分析**
- **识别逻辑**：配送费 > 6元 且 订单毛利 ≤ 配送费
- **数据来源**：订单聚合数据
- **分析维度**：
  - 高配送费订单数量
  - 配送成本总额
  - 距离分布（0-2km、2-3km、3-4km等）
  - 时段分布
- **业务影响**：配送成本侵蚀利润
- **行动建议**：优化配送范围或提高起送价
- **详情功能**：
  - 订单明细（订单时间、配送距离、配送费、订单金额）
  - 距离×时段热力图

**3. 售罄品预警**
- **识别逻辑**：库存=0 且 近7天有销量
- **数据来源**：商品库存数据 + 销售流水
- **分析维度**：
  - 售罄商品数量
  - 预估损失销售额
  - 品类分布
- **业务影响**：错失销售机会
- **行动建议**：紧急补货
- **详情功能**：
  - 商品明细（商品名称、日均销量、预估损失）

**4. 流量骤降商品**
- **识别逻辑**：销量环比下滑 > 50%
- **数据来源**：商品销售流水（7日对比）
- **分析维度**：
  - 下滑商品数量
  - 下滑幅度TOP 20
  - 下滑原因分析（突发停售、价格上涨、竞品冲击）
- **业务影响**：销售额快速下降
- **行动建议**：检查价格/库存/竞品
- **详情功能**：
  - 商品明细（商品名称、前7天销量、后7天销量、下滑率）
  - 下滑原因标签

#### 🟡 关注观察层（本周内处理）

**1. 新品滞销预警**
- **识别逻辑**：上架 < 30天 且 连续7天无销量
- **数据来源**：商品首次销售日期 + 销售流水
- **分析维度**：
  - 新品滞销数量
  - 品类分布
  - 库存占用
- **业务影响**：新品推广失败
- **行动建议**：加大营销或调整定价

**2. 价格异常商品**
- **识别逻辑**：售价 < 成本价
- **数据来源**：商品价格 + 成本数据
- **分析维度**：
  - 异常商品数量
  - 预估亏损金额
- **业务影响**：定价错误导致亏损
- **行动建议**：修正价格

**3. 利润率下滑商品**
- **识别逻辑**：当前利润率 < 历史高光利润率
- **数据来源**：商品历史利润率数据
- **分析维度**：
  - 下滑商品数量
  - 利润率差距
  - 下滑原因（成本上涨、价格下降）
- **业务影响**：盈利能力下降
- **行动建议**：分析成本或优化定价

#### 🟢 正向激励层（今日亮点）

**1. 高利润商品榜**
- **识别逻辑**：利润额 TOP 20
- **数据来源**：商品销售数据
- **展示内容**：
  - 商品名称
  - 利润额
  - 利润率
  - 销量
- **业务价值**：识别明星商品，加大推广

**2. 高利润率商品榜**
- **识别逻辑**：利润率 TOP 20
- **数据来源**：商品销售数据
- **展示内容**：
  - 商品名称
  - 利润率
  - 销量
  - 销售额
- **业务价值**：发掘潜力商品

## Tab 1: 订单数据概览

### 功能定位
**经营健康度仪表盘** - 全面展示门店核心经营指标和趋势

### 核心功能模块

#### 1. 整体指标卡片（6个核心KPI）

**📦 订单总数**
- **计算公式**：`len(order_agg)`
- **环比对比**：当前周期 vs 上一等长周期
- **上周同期**：当前周期 vs 上周同期（7天前）
- **业务含义**：订单量是业绩的基础指标

**💰 商品实收额**
- **计算公式**：`SUM(实收价格)`
- **环比对比**：✅
- **上周同期**：✅
- **业务含义**：消费者实际支付金额，反映真实购买力

**💎 总利润**
- **计算公式**：`SUM(订单实际利润)`
- **利润公式**：`利润额 - 平台服务费 - 物流配送费 + 企客后返`
- **环比对比**：✅
- **上周同期**：✅
- **业务含义**：真实净利润，扣除所有成本

**🛒 平均客单价**
- **计算公式**：`实收价格 / 订单数`
- **环比对比**：✅
- **上周同期**：✅
- **业务含义**：平均每单消费金额，反映客户价值

**📈 总利润率**
- **计算公式**：`总利润 / 实收价格 × 100`
- **环比对比**：✅（百分点差值）
- **上周同期**：✅
- **业务含义**：盈利能力指标

**🏷️ 动销商品数**
- **计算公式**：`COUNT(DISTINCT 商品名称) WHERE 月售 > 0`
- **环比对比**：✅
- **上周同期**：✅
- **业务含义**：有销量的SKU数量，反映商品活跃度

**环比显示规则**：
- 整周数据（7天）：只显示环比，隐藏上周同期（避免重复）
- 非整周数据：同时显示环比和上周同期

#### 2. 渠道表现对比

**核心指标**（每个渠道展示）：
- 订单数 + 环比
- 销售额 + 环比
- 总利润 + 环比
- 客单价 + 环比
- 利润率 + 环比（百分点）
- 营销成本率
- 佣金率
- 配送成本率
- 单均营销费用
- 单均配送费
- 单均利润

**渠道筛选规则**：
- 自动排除：收银机订单、闪购小程序（可配置）
- 支持自定义渠道筛选

**渠道健康度评级**：
- 🟢 优秀：利润率 > 15% 且 营销成本率 < 20%
- 🟡 良好：利润率 > 10% 或 营销成本率 < 30%
- 🔴 需改进：利润率 < 10% 且 营销成本率 > 30%

**下钻分析功能**（4层金字塔架构）：
- **第1层**：渠道概览（当前页面）
- **第2层**：点击渠道 → 渠道详情（订单分布、时段分析）
- **第3层**：点击商品 → 商品详情（销售趋势、利润分析）
- **第4层**：点击订单 → 订单明细（商品构成、成本结构）

#### 3. 客单价深度分析

**价格区间分布**：
- 0-10元
- 10-20元
- 20-30元
- 30-40元
- 40-50元
- 50-100元
- 100-200元
- 200元以上

**每个区间展示**：
- 订单数量
- 订单占比
- 销售额
- 销售额占比
- 平均利润率
- 亏损订单数
- 亏损订单占比

**四大业务价格组**：
- **流量区（<15元）**：引流低价商品
  - 业务定位：吸引流量
  - 典型商品：特价单品、促销品
  - 优化策略：控制占比，避免过度引流

- **主流区（15-30元）**：日常高频商品
  - 业务定位：主力销售
  - 特点：通常占订单量的50-60%
  - 优化策略：保持稳定，提升转化

- **利润区（30-50元）**：毛利贡献主力
  - 业务定位：利润来源
  - 特点：利润率相对较高
  - 优化策略：增加占比，提升客单价

- **高价区（≥50元）**：高端/大单商品
  - 业务定位：高价值客户
  - 特点：单量少但客单价高
  - 优化策略：精准营销，提升转化

**购物篮深度分析**：
- **计算公式**：`总商品行数 / 订单数`
- **业务含义**：平均每单包含多少个商品
- **优化方向**：通过组合销售提升购物篮深度

**渠道筛选功能**：
- 支持按渠道查看客单价分布
- 对比不同渠道的客单价结构

#### 4. 一级分类销售趋势

**展示内容**：
- TOP 20 分类（按销售额排序）
- 销售额趋势线（按周）
- 利润额趋势线
- 利润率趋势线（右Y轴）

**分析维度**：
- 分类销售额占比
- 分类利润贡献
- 分类利润率对比
- 周环比变化

**渠道筛选**：
- 支持按渠道查看分类趋势
- 对比不同渠道的品类表现

#### 5. 每日趋势分析

**三条趋势线**：
- 📦 订单量趋势（柱状图）
- 💰 销售额趋势（折线图）
- 💎 利润趋势（折线图）

**异常标注**：
- 自动识别异常波动点
- 标注异常原因（如促销活动、节假日）

**时间粒度**：
- 按日展示
- 支持日期范围筛选

#### 6. 详细分析区域（点击展开）

**包含内容**：
- 成本结构分析
- 营销活动分析
- 配送成本分析
- 业务逻辑说明

**成本结构分析**：
- 商品成本占比
- 营销成本占比
- 配送成本占比
- 平台佣金占比

**营销活动分析**：
- 满减金额分布
- 商品减免分析
- 代金券使用情况
- 营销ROI分析

**配送成本分析**：
- 配送费分布
- 配送净成本分析
- 用户支付配送费
- 配送费减免情况

```

```
1. "订单概览" - 月度业绩总结
   - 对比月度目标完成情况
   - 分析月度趋势

2. "今日必做" - 问题复盘
   - 统计问题处理情况
   - 分析问题根因

3. 制定下月经营策略
```

### 专项优化场景

#### 利润提升专项
```
目标：提升整体利润率5个百分点

步骤：
1. "订单概览" - 识别低利润率渠道
   → 降低营销成本或提高客单价

2. "今日必做" - 处理亏损订单
   → 调整满减规则
   → 优化配送范围

3. 持续跟踪利润率变化
```

#### 渠道策略优化
```
目标：优化渠道资源配置，提升ROI

步骤：
1. "订单概览" - 渠道对比分析
   → 识别高价值渠道
   → 识别低效渠道

2. 制定渠道策略
   → 高价值渠道：增加投入
   → 低效渠道：降低成本或退出

3. "今日必做" - 处理渠道问题
   → 满减穿底：调整活动规则
   → 高配送费：优化配送范围

4. 持续跟踪渠道ROI
```

### 常见问题诊断路径

#### 问题1：利润率下降
```
诊断路径：
1. "订单概览" → 查看利润率环比
2. "订单概览" → 渠道对比 → 识别低利润率渠道
3. "今日必做" → 满减穿底 → 识别亏损订单

根因可能：
- 营销成本过高
- 满减规则不合理
- 配送成本上升
- 低价商品占比增加

优化方案：
- 降低营销投入
- 优化满减规则
- 优化配送范围
- 提升客单价
```

#### 问题2：销售额下降
```
诊断路径：
1. "订单概览" → 查看销售额环比
2. "订单概览" → 渠道对比 → 识别下降渠道
3. "今日必做" → 流量骤降 → 识别问题商品

根因可能：
- 畅销商品缺货
- 竞品价格更低
- 营销活动减少
- 渠道流量下降

优化方案：
- 紧急补货畅销品
- 调整商品价格
- 增加营销活动
- 优化渠道投放
```

#### 问题3：缺货损失
```
诊断路径：
1. "今日必做" → 售罄品预警
2. "订单概览" → 查看畅销商品
3. "订单概览" → 分析销售趋势

根因可能：
- 备货不足
- 销量预测不准
- 补货周期长
- 供应链问题

优化方案：
- 紧急补货
- 优化备货策略
- 缩短补货周期
- 建立安全库存
```
