### 系统定位

**智能门店看板是一个数据驱动的门店经营诊断系统**，通过多维度数据分析，帮助运营者：
- 🔍 快速发现经营问题

- ✅ 获得可执行的优化建议
- 📈 持续跟踪改进效果

## 核心模块详解

### Tab 0: ✅ 今日必做

#### 功能定位
**问题优先级管理中心** - 自动识别最紧急的经营问题，按优先级分层展示

#### 核心功能

**1. 商品健康分析（六象限分类体系 V7.0）**

**功能定位**: 商品健康度诊断系统，通过六象限分类体系，帮助运营者快速识别不同类型商品的健康状况和优化方向。

**六大象限分类**:

| 象限 | 识别条件 | 业务含义 | 优化建议 |
|------|---------|---------|---------|
| 🎯 策略引流 | 极端低价（≤0.01元）或亏损引流（利润率<-50%）+ 有销量 | 主动引流商品，战略性亏损 | 控制引流成本，评估引流效果 |
| 🌟 明星商品 | 高利润率 + 高动销 + 高单品价值（≥0.5元或总贡献≥50元） | 核心盈利商品，量利双高 | 保持供应，可小幅提价 |
| 🔥 畅销商品 | 低价 + 高销量 + 正利润 | 刚需基础品（如包子、矿泉水） | 保证库存，稳定价格 |
| 💎 潜力商品 | 高利润率 + 低动销 + 有价值 | 待推广的利润品 | 加大营销，提升曝光 |
| ⚡ 自然引流 | 低利润率 + 高动销 | 市场验证的引流品 | 评估是否提价或淘汰 |
| 🐌 低效商品 | 低利润率 + 低动销 | 待优化或淘汰商品 | 降价促销或下架 |

**核心评分指标**:

1. **动销指数**（V7.1优化版）
   - 计算公式: 0.6×标准化销量 + 0.4×标准化订单数
   - 业务含义: 综合反映商品的销售活跃度
   - 优化点: 移除库存周转率（订单数据无法准确计算）

2. **利润指数**
   - 基于品类中位数的相对排名
   - 业务含义: 商品在同品类中的盈利能力
   - 优势: 品类内部竞争，避免跨品类误判

3. **单品价值**
   - 单品利润额: 每件商品的利润贡献
   - 总利润贡献: 商品的总体利润贡献
   - 业务含义: 防止低价高利润率商品被误判为明星

**动态阈值机制**（V7.2核心优化）:

系统自动根据门店规模调整判定标准：
- 高动销门槛: 销量≥70分位数（约前30%）
- 策略引流门槛: 销量≥50分位数（中位数）
- 保底门槛: 销量≥5件，订单≥2单

**趋势分析功能**:

支持多周期对比分析：
- 7天 vs 15天 vs 30天 vs 60天 vs 90天
- 识别商品健康度变化趋势
- 及时发现问题商品

**导出功能**:

一键导出商品健康分析报告（Excel格式），包含：
- 六象限分类
- 动销指数、利润指数、单品价值
- 销量、利润率、库存等核心指标
- 优化建议

**业务价值**:

| 价值维度 | 具体收益 |
|---------|---------|
| **商品结构优化** | 识别明星商品和低效商品，优化商品组合 |
| **库存管理** | 发现滞销品，减少库存积压 |
| **定价策略** | 识别潜力商品，制定差异化定价 |
| **营销决策** | 区分策略引流和自然引流，优化营销投入 |

**使用场景**:

**场景1: 商品结构优化**
```
目标: 优化商品组合，提升整体盈利能力

步骤:
1. 查看六象限分布
2. 识别明星商品（保持供应）
3. 识别低效商品（降价或淘汰）
```

**2. 智能调价计算器（V3.0 四大场景）**

**功能定位**: 基于数据驱动的定价决策系统，通过分析商品的销售数据、成本结构、价格弹性等多维度信息，为商家提供科学的调价建议。

**四大调价场景**:

| 场景 | 识别条件 | 调价方向 | 业务目标 |
|------|----------|----------|----------|
| 🩸 亏损止血 | 实收价格 < 真实保本价 | 涨价 | 止损，恢复到保本线以上 |
| 📈 利润修复 | 当前利润率 < 高光利润率 且 不亏损 | 涨价 | 恢复到历史最佳利润率 |
| 📦 滞销清仓 | 连续7天+无销量 且 库存>0 | 降价 | 清理库存，加快周转 |
| 🎯 促销引流 | 全部商品，用户自选 | 降价 | 吸引流量，提升销量 |

**核心功能模块**:

**1. 商品角色识别（10种角色）**

系统自动识别商品角色，提供差异化调价策略：

| 角色 | 判断条件 | 图标 | 调价建议 |
|------|---------|------|----------|
| 引流款 | 折扣率<30% 或 (营销占比>15% 且 利润率<5%) | 🚀 | 策略性低价，允许调价 |
| 特价款 | 30%≤折扣率<50% | 🏷️ | 促销商品，允许调价 |
| 亏损款 | 利润率<0（非引流/特价） | 🔴 | 必须提价止损 |
| 低频款 | 销量排名后10% | 📦 | 可调价促销 |
| 明星款 | 高盈利(>20%) + 高动销(前30%) + 低营销(<10%) | 🌟 | 可小幅提价 |
| 现金牛 | 高盈利 + 高动销 + 高营销 | 💰 | 可试探提价 |
| 走量款 | 低盈利(5-20%) + 高动销 | ⚡ | 建议提价 |
| 潜力款 | 高盈利 + 中低动销 | 💎 | 可提价 |
| 低动销款 | 低盈利(<20%) + 低动销(后30%) | 📉 | 可降价促销 |
| 正常款 | 其他 | ⚪ | 根据目标调整 |

**2. 价格弹性计算**

**价格弹性系数** = 销量变化率 / 价格变化率

弹性系数获取优先级：
1. **学习数据** 🎓: 该商品在该渠道有≥2次调价样本
2. **历史数据** 📊: 该商品在该渠道有≥3次调价记录
3. **跨渠道历史** 📊: 该商品跨渠道有≥3次调价记录
4. **品类默认** ⚠️: 根据一级分类匹配
5. **全局默认** ⚠️: -1.0

**品类默认弹性系数**（部分示例）:
- 生鲜: -1.5（高敏感，时效性强）
- 水果: -1.4（高敏感）
- 饮料: -1.2（较敏感，可替代性强）
- 零食: -1.0（中等敏感）
- 日用品: -0.6（刚需，敏感度低）
- 粮油调味: -0.5（刚需，敏感度最低）
- 母婴: -0.4（刚需，品牌忠诚度极高）

**3. 智能调价建议**

系统自动计算最优调价方案：

**计算逻辑**:
1. 计算目标价格 = 成本 / (1 - 目标利润率)
2. 边界约束:
   - 保本价（floor_price）: 真实保本价 = 成本 / (1 - 平台费率8%)
   - 最高价（ceiling_price）: 商品原价
3. 方向调整:
   - 涨价模式: 建议价格 = max(目标价格, 当前价格)，不超过原价
   - 降价模式: 建议价格 = min(目标价格, 当前价格)，不低于保本价
4. 预估变化:
   - 销量变化率 = 价格变化率 × 弹性系数
   - 利润变化 = (新价格 - 成本) × 新销量 - (当前价格 - 成本) × 当前销量

**返回结果**:
- 建议价格
- 预估销量变化
- 预估利润变化
- 风险等级（低/中/高）
- 弹性来源（学习/历史/默认）

**4. 弹性系数学习机制（V3.0新增）**

系统从实际调价效果中学习，持续优化预测准确性：

**学习原理**:
1. 调价执行: 商家根据建议调整价格
2. 效果观察: 系统监测调价后7天的销量变化
3. 弹性学习: 反推实际弹性系数
   ```
   实际弹性 = (销量变化率) / (价格变化率)
   ```
4. 加权更新:
   ```
   新弹性 = (旧弹性 × 旧样本数 + 新弹性) / (旧样本数 + 1)
   ```
5. 优先使用: 下次调价时优先使用学习到的弹性系数

**数据有效性检查**:
- 价格变化幅度 ≥ 2%（太小不可靠）
- 调价前必须有销量
- 弹性合理性: -5 < 弹性 < 1（排除异常值）

**5. 批量调价导出**

支持Excel导出，便于批量执行：
- 商品信息（店内码、商品名称、渠道）
- 当前价格、建议价格、调价幅度
- 预估销量变化、预估利润变化

**业务价值**:

**🔴 紧急处理层（今日必须完成）**
1. **满减穿底订单**
   - 识别：满减优惠 > 订单毛利的订单
   - 影响：直接亏损，每单都在赔钱
   - 行动：立即调整满减规则或商品定价

2. **高配送费订单**
   - 识别：配送费 > 6元 且 毛利无法覆盖
   - 影响：配送成本侵蚀利润
   - 行动：优化配送范围或提高起送价

3. **售罄品预警**
   - 识别：库存=0 且 近7天有销量
   - 影响：错失销售机会
   - 行动：紧急补货

4. **流量骤降商品**
   - 识别：销量环比下滑 > 50%
   - 影响：销售额快速下降
   - 行动：检查价格/库存/竞品

**🟡 关注观察层（本周内处理）**
1. **新品滞销**
   - 识别：上架 < 30天 且 连续7天无销量
   - 影响：新品推广失败
   - 行动：加大营销或调整定价

2. **价格异常**
   - 识别：售价 < 成本价
   - 影响：定价错误导致亏损
   - 行动：修正价格

3. **利润率下滑**
   - 识别：当前利润率 < 历史高光利润率
   - 影响：盈利能力下降
   - 行动：分析成本或优化定价

### Tab 1: 📊 订单数据概览

#### 功能定位
**经营健康度仪表盘** - 全面展示门店核心经营指标和趋势

#### 核心功能模块

**1. 整体指标卡片**
- 订单总数 + 环比
- 预计零售额 + 环比
- 总利润 + 环比
- 平均客单价 + 环比
- 总利润率 + 环比
- 动销商品数 + 环比

**2. 渠道表现对比**
- 多渠道业绩横向对比
- 订单数/销售额/利润/客单价/利润率
- 营销成本率/佣金率/配送成本率
- 单均营销费用/单均配送费/单均利润

**3. 客单价深度分析**
- 价格区间分布（0-10元、10-20元...）
- 四大业务价格组分析
  - 流量区（<15元）：引流低价商品
  - 主流区（15-30元）：日常高频商品
  - 利润区（30-50元）：毛利贡献主力
  - 高价区（≥50元）：高端/大单商品
- 购物篮深度分析

**4. 一级分类销售趋势**
- 品类销售额周趋势
- 品类占比分析
- 环比变化追踪

**5. 每日趋势图表**
- 订单量/销售额/利润的时间序列
- 趋势线和波动分析

#### 业务价值

| 价值维度 | 具体收益 |
|---------|---------|
| **全局掌控** | 一屏看清门店整体经营状况 |
| **趋势洞察** | 及时发现业绩波动和异常 |
| **渠道优化** | 识别高价值渠道，优化资源投入 |
| **定价策略** | 通过客单价分析优化商品组合 |

**场景2：渠道策略优化**

### 问题诊断流程

```
发现问题 → 定位根因 → 制定方案 → 执行优化 → 跟踪效果
   ↓          ↓          ↓          ↓          ↓
今日必做   订单概览   制定方案   执行动作   持续监控
```

### 常见问题诊断路径

#### 问题1：利润率下降
```
诊断路径：
1. "订单概览" → 查看利润率环比
2. "订单概览" → 渠道对比 → 识别低利润率渠道
3. "今日必做" → 满减穿底 → 识别亏损订单

根因可能：
- 营销成本过高
- 满减规则不合理
- 配送成本上升
- 低价商品占比增加

优化方案：
- 降低营销投入
- 优化满减规则
- 优化配送范围
- 提升客单价
```

#### 问题2：销售额下降
```
诊断路径：
1. "订单概览" → 查看销售额环比
2. "订单概览" → 渠道对比 → 识别下降渠道
3. "今日必做" → 流量骤降 → 识别问题商品

根因可能：
- 畅销商品缺货
- 竞品价格更低
- 营销活动减少
- 渠道流量下降

优化方案：
- 紧急补货畅销品
- 调整商品价格
- 增加营销活动
- 优化渠道投放
```

#### 问题3：缺货损失
```
诊断路径：
1. "今日必做" → 售罄品预警
2. "订单概览" → 查看畅销商品
3. "订单概览" → 分析销售趋势

根因可能：
- 备货不足
- 销量预测不准
- 补货周期长
- 供应链问题

优化方案：
- 紧急补货
- 优化备货策略
- 缩短补货周期
- 建立安全库存
```

### 核心指标定义

| 指标 | 计算公式 | 业务含义 |
|------|---------|---------|
| 客单价 | 实收价格 / 订单数 | 平均每单消费金额 |

| 配送成本率 | 配送净成本 / 销售额 × 100 | 配送支出占比 |
| 库存周转天数 | 当前库存 / 日均销量 | 库存消耗速度 |
| 动销率 | 有销量商品数 / 总商品数 × 100 | 商品活跃度 |
