### 4.2 订单数据不一致 ✅ 已修复

**问题描述**:
- Vue版本显示灵璧县门店订单总数: 5,847笔
- Dash版本显示灵璧县门店订单总数: 2,771笔
- 差异: 约2倍

**根本原因**:
通过诊断脚本分析发现:
- Vue版本初始化时`dateRange`为空数组，导致API请求没有传递日期参数
- 后端返回了**全部日期**的数据（5,847笔）
- Dash版本默认加载**最近30天**的数据，用户当前选择的是约**最近14天**（2,679笔，接近2,771）

**诊断数据**:
```
数据库灵璧县门店数据:
- 日期范围: 2025-11-04 ~ 2025-12-03
- 全部日期订单数: 5,847
- 最近7天订单数: 1,382
- 最近14天订单数: 2,679 (接近Dash版本的2,771)
- 最近30天订单数: 5,847
```

**修复方案**:
修改 `frontend/src/views/OrderOverview.vue`，在初始化时设置默认日期范围为最近30天:

```typescript
// 初始化默认日期范围（最近30天，与Dash版本一致）
const getDefaultDateRange = (): string[] => {
  const end = new Date()
  const start = new Date()
  start.setDate(start.getDate() - 29) // 最近30天
  
  const formatDate = (d: Date) => {
    const year = d.getFullYear()
    const month = String(d.getMonth() + 1).padStart(2, '0')
    const day = String(d.getDate()).padStart(2, '0')
    return `${year}-${month}-${day}`
  }
  
  return [formatDate(start), formatDate(end)]
}

const dateRange = ref<string[]>(getDefaultDateRange())
```

### 5.3 缓存策略优化 ✅ 已完成 (2026-01-06)

**缓存架构**:
```
┌─────────────────────────────────────────────────────────┐
│                    缓存层级                              │
├─────────────────────────────────────────────────────────┤
│  L1: 内存缓存 (_memory_cache)                           │
│      - store_cache: {门店名: {data: df, timestamp: t}}  │
│      - 访问速度: 最快                                    │
├─────────────────────────────────────────────────────────┤
│  L2: Redis缓存                                          │
│      - key: order_data:{门店名}                         │
│      - TTL: 300秒 (5分钟)                               │
│      - 访问速度: 快                                      │
├─────────────────────────────────────────────────────────┤
│  L3: PostgreSQL数据库                                   │
│      - 按门店查询: WHERE store_name = ?                 │
│      - 访问速度: 较慢                                    │
└─────────────────────────────────────────────────────────┘
```

**缓存命中流程**:
1. 检查内存缓存 → 命中则返回
2. 检查Redis缓存 → 命中则返回并更新内存缓存
3. 查询数据库 → 更新Redis和内存缓存

**性能提升**:
- 首次加载: 从数据库查询 (约2-5秒)
- 缓存命中: 从内存返回 (约0.01秒)
- 预期提升: 100-500倍
